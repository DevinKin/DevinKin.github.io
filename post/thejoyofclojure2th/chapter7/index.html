第7章-函数式编程

各种形式的函数

函数是一等公民

一等的概念

  • 他可以根据需要创建
  • 他可以作为数据结构存储
  • 他可以作为参数传递给函数
  • 他可以作为函数的返回值

comp 函数接受函数列表, 并将函数组合.

partital 函数创建一个接受部分参数的新函数(偏函数).

一个使用 partital 创建的函数试图在给定另一个参数时求值. 一个局部函数返回另一个局部函数, 直到他接受到预订数量的参数.

complement 函数接受一个返回真值的函数并且返回与之相反的真值.

高阶函数

高阶函数是一个至少具有以下几点的函数

  • 接受一个或多个函数作为参数
  • 将函数作为返回值返回

纯函数是常规函数, 符合以下规则

  • 函数给定相同的参数总是返回相同的结果
  • 函数不会产生任何可观察的副作用

Clojure的每一个函数都有可能收到其输入和输出方面的约束, 以及他们之间任意关系的约束. 函数的约束映射有两项

  • :pre 限制输入参数
  • :post 限制返回值
  • 可以通过 (set! *assert* false) 关闭前置和后置校验

闭包

闭包就是拥有其被创建的局部上下文访问权限一个函数.

递归

普通递归

普通递归方式容易造成栈溢出, 可以使用尾递归方式解决

(defn pow [base exp]
  (letfn [(kapow [base exp acc]
	    (if (zero? exp)
	      acc
	      (recur base (dec exp) (* base acc))))]
    (kapow base exp 1)))

lz-rec-step 使用常规递归但是使用了 lazy-seq 避免的栈溢出异常.

尾递归

TCO(tail call optimization) 尾递归优化.

由于Clojure和JVM都不支持尾递归优化, Clojure只支持自身调用的尾递归, 对于广义的尾递归并不支持.

由于JVM不支持尾递归的优化, 所以使用 recur 时要注意避免栈溢出错误.

尾部位置和递归目标

形式 尾部位置 递归目标
fn, defn (fn [args] expressions tail) 是
loop (loop [bindings] expressions tail) 是
let, letfn, bindings (let [bindings] expressions tail) 否
do (do expressions tail) 否
if, if-not (if test then-tailelse-tail) 否
when, when-not (when test expressions tail) 否
cond (cond test test tail … :else else tail) 否
or, and (or test test … tail) 否
case (case const const tail … default tail) 否

Clojure强制要求 recur 调用需要在尾部位置.

trampoline 函数解决了互相递归调用函数的问题.

continuation-passing style(CPS) 是递归和互相递归的混合品.
